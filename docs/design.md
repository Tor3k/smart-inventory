# Smart Inventory Design

## Project Goal

Smart Inventory is an inventory management system designed for a small bakery. The project is being developed incrementally, focusing on clean architecture, maintainability, and future scalability.

---

## Product Types

### Purchased Products

- Purchased from suppliers.
- Have a purchase cost.
- May include a brand.
- Can be resold.

### Manufactured Products

- Produced locally.
- Do not require a brand.
- Have production costs calculated separately (future feature).

---

## Stock Rules

- Stock cannot become negative.
- Products are never permanently deleted.
- Products can be marked as inactive.
- Every stock movement should be traceable (future feature).

---

## Current Architecture

- Console application
- Object-Oriented Programming
- Modular design
- English code / Spanish user interface
